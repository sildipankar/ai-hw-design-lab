# Common IP Narrative — the parts bin in plain words (companion to 01_common_ip.md)

Read this like a tour of a toolbox. Format follows ai-hw-os/01_architecture_mental_models/design_narrative_template.md, adapted for a library: there is no top level and no single transaction, so each block gets a worked micro-example with real values instead.

## The whole library in three sentences
These are the nine building blocks everything else in AXION is assembled from: two pattern engines (make random words, compress results to a fingerprint), three traffic movers (buffer in one clock, buffer across two clocks, pipeline-register a handshake), one referee (round-robin arbiter), two CDC couriers (level and pulse synchronizers), and clock/reset plumbing. Every block obeys the same house rules: one clock in, synchronous active-high `rst` inside every flop, and after reset it sits perfectly still until you deposit an enable, a valid, or a load — nothing free-runs. "Working" means each block passes its own standalone Protium bring-up row (bottom of file 01) before anything is allowed to instantiate it.

**The shared reset story:** there is no POR sequencing here — each block clears its own state on `rst` (LFSR to 32'h1, MISR to 0, FIFOs to empty, arbiter pointer to 0, CRC to 0xFFFF) and then waits. The one block that CREATES clean resets for everyone else is `reset_sync`; the one rule that spans two blocks is that `async_fifo`'s two resets must be asserted together (the SoC owns that).

## C-01 lfsr32 — the die that always rolls the same way
A seeded pseudo-random word source: one 32-bit state register and the XOR taps 0x80200003 (Galois, x^32+x^22+x^2+x^1+1). **Logic in one sentence:** on `load`, state takes `seed` (a seed of 0 is silently replaced by 32'h1 so the LFSR can't lock up in the all-zero state); on `enable`, shift right one and XOR in the taps if the bit that fell off was 1. **Signals in words:** `load` wins over `enable`; `prdata` is the state itself (reset value 32'h1, never 0 afterwards); `valid` is just `enable` registered — "I advanced last cycle." SPEC GAP: whether `valid` asserts on a cycle where `load` overrode `enable` — "registered enable" says yes, "one cycle per advance" says no. **The one subtlety:** shift is to the RIGHT and taps apply when the LSB was 1 — a model that shifts left produces a valid-looking but wrong sequence that only the sim-vs-HW compare catches. **Watch:** prdata is never 0x0. **Micro-example:** seed 0x1 → 0x80200003 → 0xC0300002 → 0x60180001 (step 1: 0x1>>1=0, LSB was 1, XOR taps; step 3: LSB was 0, pure shift). You can check those three by hand.

## C-02 misr32 — the fingerprint jar
A signature compressor: everything that ever entered leaves a trace, and the final 32-bit `signature` is meaningful only compared against the same stream replayed in sim. **Logic in one sentence:** each `valid` cycle, shift the signature left one, feed back sig[31]^sig[21]^sig[1]^sig[0] into the new bottom bit, then XOR the whole thing with `data`. **Signals in words:** `clear` — zero the signature mid-run without touching anything else (state effect identical to `rst`). SPEC GAP: priority when `clear` and `valid` land on the same cycle — is that word absorbed or discarded? **The one subtlety:** order matters — the same words in a different order give a different signature; that is the feature (it catches reordering), but it means the sim and HW stimulus must be bit-and-cycle identical. **Watch:** signature after the 16-word bring-up pattern (1,2,4,…,0x8000) equals the sim-logged value. **Micro-example:** from 0, feed 0x1, 0x2, 0x4 → signature goes 0x1, 0x1, 0x7. The middle step is the eye-opener: shift-with-feedback turns 0x1 into 0x3, and XOR with data 0x2 lands it back on 0x1.

## C-03 sync_fifo — the single-clock waiting room
A 16-deep show-ahead FIFO: whenever it isn't empty, `rdata` is already showing the oldest word — `rd_en` doesn't fetch, it discards-and-advances. **Logic in one sentence:** push on `wr_en` unless full, pop on `rd_en` unless empty, and both refusals are silent (the request is simply ignored — no error flag). **Signals in words:** `count` — occupancy, readable at the bench; `full`/`empty` — registered flags from the AW+1-bit pointer compare (equal = empty, equal-except-MSB = full). **The one subtlety:** FWFT plus silent-drop — a write offered while full vanishes without a trace even if a pop happens the same cycle, so upstream must respect `full`, always. **Watch:** count breathes up and down and lands on 0 at idle; total pushes minus total pops must equal count at any instant. **Micro-example:** push 0xA0,0xA1,0xA2,0xA3 → count reads 4 and rdata is ALREADY 0xA0 with no read issued; four pops return A0,A1,A2,A3 and `empty` re-asserts.

## C-04 rr_arbiter — the referee with a rotating memory
Four requesters, one winner per cycle, fairness by remembering who won last. **Logic in one sentence:** search for the first request starting one past the last winner (the `{req,req} >> (last+1)` double-vector trick), register the one-hot grant, and move the pointer only on cycles where a grant was actually issued. **Signals in words:** `grant` — one-hot and registered, so it shows up one cycle after the requests it answers; `grant_id` — the same winner in binary; `grant_valid` — "somebody won." Reset puts the priority pointer at 0, grants low. **The one subtlety:** the grant is a *statement*, not a handshake — there is no accept signal, so the pointer advances on issue; and because grant is registered, a requester that drops `req` at the wrong moment can be shown a one-cycle grant it no longer wants. SPEC GAP: required behavior when `req` deasserts in the cycle its grant is being registered. **Watch:** grant one-hot every cycle, and with req=4'b1111 the id rotates 0,1,2,3,0… **Micro-example:** last winner = 1, req = 4'b1001 → search order 2,3,0,1 → grant = 4'b1000 (id 3); req still 1001 next cycle → grant = 4'b0001 (id 0). With req = 4'b0101 forever, id alternates 0,2,0,2.

## C-05 skid_buffer — the one-chair anteroom
A pipeline stage that lets you cut a long ready wire without losing throughput: main register plus one spare "skid" slot. **Logic in one sentence:** pass words through at full rate, and when downstream stalls, catch the one extra word that upstream was already committing (because our registered `s_ready` tells it "go" one cycle late) in the skid slot. **Signals in words:** `s_ready` — registered, which is the entire point: no combinational path from `m_ready` back to the source; `m_valid`/`m_data` — registered outputs. Reset: empty, s_ready will assert, m_valid low. **The one subtlety:** the late `s_ready` means one in-flight word ALWAYS arrives after a stall begins — the skid slot isn't an optimization, it's the only thing preventing that word's loss. **Watch:** with incrementing s_data and m_ready toggling ~50%, m_data must count with no skip and no repeat. **Micro-example:** stream 0x10,0x11,0x12 with m_ready dropping for one cycle just as 0x11 launches: 0x11 parks in the skid slot, s_ready falls the NEXT cycle, and when m_ready returns you get 0x11 then 0x12 — nothing lost, nothing doubled.

## C-06 crc16_gen — the word notary
CRC-16-CCITT (poly 0x1021, init 0xFFFF, no reflection) eating a whole 32-bit word per cycle through an unrolled MSB-first next-state function. **Logic in one sentence:** on `init` go back to 0xFFFF; on `valid` fold `data` into the running CRC and register it. **Signals in words:** `crc` — the current remainder, reset/init value 0xFFFF; the same module is a generator in TX and a checker in RX (recompute and compare, per the d2d spec). SPEC GAP: `init` and `valid` on the same cycle — absorb into a fresh 0xFFFF, or drop the word? The d2d "init per flit" usage makes this corner real. **The one subtlety:** bits enter MSB-first — a reflected (LSB-first) implementation passes every "it toggles" smoke test and fails every real compare. **Watch:** the one-word bring-up: init, then one valid of 0xDEADBEEF. **Micro-example:** crc after init = 0xFFFF; after absorbing 0xDEADBEEF, crc = 0x4097 (and if the d2d folds it to a nibble: 4^0^9^7 = 0xA).

## C-07 sync_2ff — the two-flop border guard
Two back-to-back flops (`ASYNC_REG` on both) in the destination clock, nothing else. **Logic in one sentence:** sample the foreign signal twice so metastability has a full cycle to die. **The one subtlety:** it is only legal for quasi-static or gray-coded inputs — feed it a multi-bit binary counter and different bits resolve in different cycles, delivering a value that never existed. **Watch:** q follows d exactly 2 clk_dst edges later, every time. **Micro-example:** d rises just before a clk_dst edge → q is high two edges later; d was 0 at reset, so q wakes at 0.

## C-07 pulse_sync — the flag semaphore between worlds
Carries a single-cycle event from one clock to the other: a toggle flop in the source domain flips per pulse, sync_2ff carries the toggle level across, and an edge detector in the destination turns each flip back into one clean pulse. **Signals in words:** `pulse_in` — one src-clock event; `pulse_out` — exactly one dst-clock cycle per event, never zero, never two. **The one subtlety:** it carries EDGES, not counts — two input pulses closer together than the toggle can be resolved at the destination (about two dst clocks plus settling) merge into zero or one output. SPEC GAP: the minimum pulse spacing is not stated; in AXION it survives because the credit pulses travel clk_b→clk_a and clk_a is the fast clock. **Watch:** count pulses on both sides over a run — they must be equal. **Micro-example:** one clk_b `pulse_in` flips the toggle 0→1; two-to-three clk_a edges later, one single-cycle `pulse_out`.

## C-08 async_fifo — the airlock (the CDC workhorse)
The only legal way for DATA to change clock domains in AXION: a 16-deep dual-port RAM with a write pointer in wclk-land, a read pointer in rclk-land, and each pointer visiting the other side only as a gray code through a sync_2ff. **Logic in one sentence:** full and empty are computed from the LOCAL pointer versus the SYNCED (two-cycle-old) foreign pointer — gray-coded so a mid-flight sample can only ever read as the old or the new value, both safe. **Signals in words:** `wfull` — wclk's belief it must stop (equal gray pointers with top two bits inverted); `rempty` — rclk's belief there's nothing yet (equal gray pointers); both registered, both reset asserted-safe (empty=1, full=0). Both resets must be asserted together at system level — releasing one domain alone leaves the two sides believing different pointer histories. **The one subtlety:** the flags are pessimistic, never optimistic — wfull can stay up a couple of cycles after space exists and rempty after data exists, because the foreign pointer is stale; that costs bandwidth, never correctness. SPEC GAP: `rdata` timing — "registered output OK" leaves open whether rdata is show-ahead when !rempty or appears one cycle after rd_en; the d2d_rx that consumes it needs this pinned down. **Watch:** at 2:1 clocks, rempty deasserts within ~3 rclk of the first write and no flag ever glitches. **Micro-example:** four writes take the 5-bit write pointer binary 3→4, which in gray is 00010→00110 — exactly one bit changed, which is the whole safety argument in one transition; reads then return the four words in write order.

## C-09 clk_div — the divider you must not use
A toggle-flop divide-by-2 (counter-compare for bigger even ratios), 50% duty, clock driven from a flop. **The one subtlety is the block's entire identity:** on Protium/Palladium, clocks are tool-generated in the IXCOM clock spec — this module is EXCLUDED from all Protium filelists and exists only for a future raw-FPGA port. If clk_div ever appears in a Protium build, the build is wrong. **Watch (FPGA-only):** clk_out at half the rate, edges only on clk_in rising edges. **Micro-example:** DIV=2 — the flop toggles every clk_in edge, giving one clk_out period per two clk_in periods.

## C-09 reset_sync — the reset conditioner
Async assert, sync de-assert: a 4-flop shift register with async preset per clock domain. **Logic in one sentence:** the moment `arst_n_in` drops, `rst_out` snaps high with no clock needed; when it releases, ones drain through four flops so rst_out falls exactly 4 clk edges later, synchronous and clean. **Signals in words:** note the polarity flip — input is async active-LOW, output is the house-standard sync active-HIGH `rst`. **The one subtlety:** de-assertion is the dangerous edge (an async release near a clock edge would let half the flops leave reset a cycle apart) — that is what the four flops exist for; assertion is free. **Watch:** the bring-up row is literal: release arst_n_in at runtime, see one clean rst_out falling edge exactly 4 clks later. **Micro-example:** arst_n_in rises between clk edges N and N+1 → rst_out falls at edge N+4.

## Bench identities (library-wide freebies)
lfsr32: prdata ≠ 0, ever. sync_fifo: pushes − pops = count, and count = 0 at idle. rr_arbiter: grant one-hot always; over any window with all req high, each id wins within ±1 of the others. pulse_sync: pulses in = pulses out. misr32/crc16: replaying the identical stream in sim and HW gives the identical signature — that identity IS the validation method.
