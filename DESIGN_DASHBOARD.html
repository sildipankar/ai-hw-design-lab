2. ALINK AXI4-Lite Systems

💡 System 1 & 2 Technical Brief: AXI4-Lite Integration

The ALINK series is a single-clock control and memory subsystem designed to validate AXI4-Lite communication channels, custom memory mapping, and protocol conformance testing in emulation. It splits the design process into two manageable, fully automated system-level milestones:

  • System 1 (ALINK-Mini — Register Loopback): Verifies a direct master-to-slave link. The master chiplet hosts a register-oriented generator (cmd_gen_mini) driving the AXI Master FSM (axm_engine). It sends AXI transactions directly to the slave chiplet's register file (axs_regs) to perform write and read validation loops, compressing responses into a MISR checksum. Purpose: Proves the basic channel FSM handshake and CSR updates run with 0 errors.
  • System 2 (Full ALINK — Decoded Multi-Slave): Introduces the full memory mapping path. It adds a 1-to-2 address decoder (axs_dec) that routes read/write transactions based on address bit 15, dividing traffic between registers and an SRAM memory wrapper (axs_mem). It also incorporates a protocol monitor (axil_pmon) to flag combinational bus violations. Purpose: Validates address-range decoding, dual-port block SRAM accesses, and system-level test sweeps.

Hardware Interface: A single external go pulse triggers the automated commander FSM. The system sweeps through the write/read test phases and latches a done flag alongside a 32-bit MISR chk_sig. Verification relies on matching this signature against a golden reference.

System 1: ALINK-Mini (AXI Register Loopback with Chiplets) Master-Slave Chiplet Link

ALINK-Mini is a simplified 2-chiplet validation setup. It connects the Master Chiplet directly to the Slave Chiplet over an AXI4-Lite bus interface, bypassing the address decoder and memory blocks. To drive the transactions, the Master Chiplet contains an automated generator FSM (`cmd_gen_mini`) that runs only the register checks, enabling a self-checking hardware validation loop.

Detailed Block Descriptions

cmd_gen_mini RTL (System 1 Driver)

The automated master driver for System 1. Instantiates the prerequisite LFSR and MISR blocks. On receiving a top-level `go` pulse, it drives register writes and reads directly to check the register bank, then captures the checksum signature.

axm_engine RTL (AL-02)

Acts as the bus master protocol translator. Translates command requests into standard AXI4-Lite handshake channels (AW, W, B, AR, R), and checks for transaction timeouts.

axs_regs RTL (AL-07)

The register file slave. Resolves AXI reads and writes to ID register (0x00), scratch pads (0x04, 0x08), and a transaction success counter WRCNT (0x0C).

System 2: Full ALINK (Decoded Multi-Slave Link) Decoded Chiplets

This is the complete system. It introduces the AXI address decoder (`axs_dec`), wraps the Master and Slave components inside chiplet shells, instantiates an SRAM memory bank block, and hooks up the automated 3-phase test supervisor FSM.

Detailed Block Descriptions

cmd_gen RTL (AL-01)

A 3-phase self-test commander. It generates write traffic (Phase 1) and read traffic (Phase 2) to SRAM using internal LFSRs, then verifies status registers (Phase 3). It monitors error flags and generates a cumulative signature output.

axil_pmon RTL (AL-03)

A synthesizable protocol checker that monitors AXI channels combinationally. It validates that signals like `awvalid` stay high until `awready` asserts, ensuring protocol compliance.

axs_dec RTL (AL-06)

A 1-to-2 address decoder. Directs write and read transactions to either the Register file (if address MSB is 0) or the Memory wrapper (if address MSB is 1).

axs_mem RTL (AL-08)

Adapts the simple dual-port SRAM bank wrapper into an AXI4-Lite memory slave, mapping read/write channels to physical memory controls.

1. AXION SoC (System 2 - The Main SoC)

💡 System 2 Technical Brief: Multi-Clock Compute/DMA & CDC Link

The AXION SoC is a dual-chiplet, multi-clock domain hardware accelerator. It models a high-performance computation engine connected to a memory sub-system, illustrating asynchronous clock-domain crossing (CDC) and credit-based flow control interfaces:

  • Compute Chiplet (CCX): Operates on a high-frequency system clock (clk_a). It features a 4x4 systolic matrix-multiply array (16 identical MAC processing elements) fed by an internal sequencer FSM. Computations are gathered by a reduction accumulator, formatted, and pushed to the Die-to-Die Link.
  • Die-to-Die Link (D2D): Manages the asynchronous boundary. d2d_tx aggregates computed results, encapsulates them with a sequence index and CRC-16 checksum, and writes them to a dual-clock asynchronous FIFO. d2d_rx reads from the FIFO on clk_b, verifies frame checksums, and returns buffer-avail credit pulses back to the fast domain via toggle-based synchronizers.
  • Memory/IO Chiplet (MIO): Operates on a slower, half-frequency clock (clk_b). A multi-channel DMA Move Engine unloads packets from D2D, arbitrates access through a round-robin scheduler, and writes data to 4 distinct SRAM banks.

Hardware Test Loop: A top-level stimulus generator drives inputs into the compute domain based on seed settings. At the memory output, a 32-bit signature compressor gathers the final written results. This allows developers to prove cycle-and-data equivalence across the entire asynchronous boundary.

AXION SoC Top-Level Architecture Dual Clock Domain (CDC)

AXION is a dual-chiplet accelerator SoC designed to run on Cadence Protium. The Compute Chiplet (CCX) runs on `clk_a` and the Memory/IO Chiplet (MIO) runs on `clk_b` (half speed). They communicate across an asynchronous boundary via the synthesizable Die-to-Die Link (d2d_link).

Detailed Block Descriptions

Here is the detailed function of every block inside the AXION design:

lfsr32 Leaf Block (C-01)

A 32-bit seeded Galois Linear Feedback Shift Register (LFSR) using tap polynomial x^32 + x^22 + x^2 + x^1 + 1. Generates pseudo-random, seed-driven stimulus words. Avoids locking up by substituting 32'h1 if a seed of 0 is loaded.

misr32 Leaf Block (C-02)

A Multiple-Input Signature Register (MISR) that compresses an incoming stream of 32-bit data words into a single 32-bit signature value. Used as the signature result checker core on hardware.

sync_fifo Leaf Block (C-03)

A 16-deep single-clock first-word-fall-through (FWFT) FIFO buffer. Holds valid data elements combinationally on its read port before a read pulse is supplied, allowing smooth pipeline handshakes.

rr_arbiter Leaf Block (C-04)

A rotating-priority round-robin arbiter for N requesters. Uses a double-request-vector shift operation to assign one-hot grants in a single clock cycle, updating priority state only when a grant is accepted.

skid_buffer Leaf Block (C-05)

A full-throughput pipeline register with a 1-slot "skid" buffer. Breaks combinational path loops on the ready/valid handshake signals without dropping or duplicating data during stall cycles.

crc16_gen Leaf Block (C-06)

Computes a CRC-16-CCITT checksum over parallel 32-bit words per cycle using an unrolled loop. Used to check data integrity of packet payloads.

sync_2ff / pulse_sync Leaf Block (C-07)

Synchronizes signals across asynchronous boundaries. `sync_2ff` passes quasi-static signals through 2 back-to-back flops. `pulse_sync` uses toggle registers to safely carry single-cycle pulse events across clock edges.

async_fifo Leaf Block (C-08)

A dual-clock Gray-pointer asynchronous FIFO buffer. Uses Gray-coded write and read pointers crossed into destination domains via sync_2ff synchronizers to pass data words safely between clocks.

clk_div / reset_sync Leaf Block (C-09)

`clk_div` is a simple clock divider (excluded in emulator builds). `reset_sync` provides asynchronous assertion and synchronous deassertion of resets to prevent reset metastability.

pe_mac CCX Leaf Block

A multiply-accumulate processing element. Performs 8-bit multiplication of incoming coefficients and adds to a running accumulator, featuring a sticky saturation flag.

pe_array CCX Sub-Unit

A 4x4 systolic matrix array grid consisting of 16 identical `pe_mac` modules to accelerate parallel arithmetic calculations.

accum_unit CCX Sub-Unit

Gathers raw output values from the systolic grid array, reduces/accumulates them, and formats them for memory storage.

seq_fsm / ccx_csr CCX Controller

`seq_fsm` orchestrates execution cycles. `ccx_csr` serves as the software mailbox interface (control and status registers).

sram_bank MIO Leaf Block

A single 1024x32-bit dual-port static RAM bank configured for reading and writing computed results or test sequences.

bank_arb MIO Sub-Unit

Uses the round-robin arbiter (`rr_arbiter`) to coordinate read/write conflicts from multiple masters trying to access the SRAM banks.

addr_gen / move_engine MIO DMA Engine

`addr_gen` handles memory read/write address pointers. `move_engine` serves as the DMA controller, transferring blocks of results autonomously.

d2d_tx D2D Link Block

Packetizes raw 32-bit word streams into packet flits containing headers, sequences, data payloads, and CRC16 checksums.

d2d_rx D2D Link Block

Unpacks data flits, verifies the CRC16 field, drops/corrupts invalid frames, and coordinates credit-based backpressure back to D2D TX.

3. SLINK (Serial Telemetry Link)

SLINK Block Layout UART Loopback Setup

SLINK routes words from a local FIFO through a serializer, down a 1-wire UART path, into an oversampling receiver, and reassembles them while validating a checksum. It operates on a single clock with internal baud ticks.

Detailed Block Descriptions

pkt_framer RTL (SL-01)

A state machine that reads 32-bit values from the FIFO and splits them into bytes. It bundles them into a packet structure: Start of Frame (0xA5) + Byte Count (4) + 16 payload bytes (4 words x 4 bytes) + XOR Checksum byte.

pkt_mon RTL (SL-02)

A performance and error checking block. It counts sent bytes, registers FIFO write errors (write attempts when full), and tracks the longest cycle stretch where downstream was stalled.

baud_tick RTL (SL-05)

A programmable count-down clock enable pulse generator. Outputs a single-cycle tick active pulse at 16x the configured baud rate, serving as the timing reference for the UART modules.

uart_tx RTL (SL-06)

Converts parallel 8-bit bytes into serial UART frames: 1 start bit, 8 data bits (LSB-first), and 1 stop bit, driving the output line `txd` high when idle.

uart_rx RTL (SL-07)

Filters noise on `rxd`. When it detects a falling start edge, it aligns to the center of the start bit, then samples subsequent bits every 16 ticks, outputs the byte, and checks for framing errors.

depack RTL (SL-08)

Reconstructs words. It scans the incoming bytes for the 0xA5 SOF, extracts the count, reconstructs four 32-bit words, validates the XOR checksum byte, and pipes the output to a checker MISR signature register.

NEXA Mesh Interconnect Packet Switched Router Grid

NEXA interconnects four nodes in a 2x2 grid. Routers use XY-routing rules to pass single-flit packets combinationally through input ports to targeted output ports.

Detailed Block Descriptions

nx_in_port RTL (NX-01)

Wraps a `skid_buffer` for ingress. It decodes the destination coordinate (X, Y) bits in the flit and decodes them to generate a one-hot request map pointing to North (0), South (1), East (2), West (3), or Local (4) output channels.

nx_out_port RTL (NX-02)

Monitors output buffers. Uses a 5-input `rr_arbiter` to choose which input port has access to its channel, extracts the target flit through a multiplexer, and registers the output handshake lines.

nx_router Structural (NX-03)

A 5-port structural router core. Interconnects 5 input ports and 5 output ports by wiring up the transposed request/grant buses and routing input flits to the multiplexers.

nx_tgen / nx_tsink Traffic Engines (NX-04 / NX-05)

`nx_tgen` acts as a local generator generating packets with destination addresses. `nx_tsink` verifies arrival indices, counts packets, and registers payload signatures.

EQD Chiplet Architecture CDC Boundary Checker

EQD is a purpose-built arithmetic datapath designed to test sequence equivalence. It can be compiled as either a direct monolithic pipeline or as a chiplet design containing a dual-clock D2D FIFO CDC boundary between eqa_top and eqb_top.

Detailed Block Descriptions

tgen RTL Generator

Generates self-identifying test words packed with stream IDs and sequence numbers. Calculates `data = (seq + SEED) & 0xFFFF` so the value can be computed in your head at the bench.

gen_arb RTL Arbiter

A 3-input round-robin arbiter that multiplexes the output words from the three generator streams into a single output pipeline path.

add_const / sub_const RTL Pipes

`add_const` adds constant `K1` to the data field of the payload on Chip A. `sub_const` subtracts `K1` on Chip B, restoring the original value.

acc_a / acc_b Accumulators

Accumulators summing up transaction payloads on both sides. Compares are clean: at completion, `acc_a - acc_b` must equal the total transfer count multiplied by K1.

ram_fifo RTL Buffer

A 16x32 simple static RAM wrapper implementing read and write pointer structures to buffer data at the boundary interface.

chk_unit RTL Verification Unit

Recomputes the expected payload on-the-fly from the sequence bits, compares it with the output from `sub_const`, and counts total successes/failures.
